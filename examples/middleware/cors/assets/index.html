Test CORS Server Endpoints

GET /simple

Will not require a pre-flight request due to being a simple method.

POST /simple

Will not require a pre-flight request since this is a simple method with a simple content type (text/plain).

PATCH /update

Will require a pre-flight request which will be blocked due to the CORS config not accepting PATCH requests.

POST /create

Will require a pre-flight request due to the request requiring a non-simple content type (application/json).

DELETE /remove

Will require a pre-flight request which will be accepted.
